\section{Introduction}
The Hilbert's Grand Hotel Management System is a comprehensive web-based application designed to streamline hotel operations. The system enables users to browse available rooms, book stays, and manage their profiles, while providing an administrative dashboard to manage hotel inventory.

\section{Technology Stack}
\begin{itemize}
    \item \textbf{Frontend:} Next.js (React), Tailwind CSS
    \item \textbf{Backend:} Next.js API Routes (Node.js)
    \item \textbf{Database:} MongoDB Atlas
    \item \textbf{Authentication:} Custom Session Management with Cookies, bcryptjs for password hashing.
\end{itemize}

\section{Database Design and Architecture}
The system uses a NoSQL database (MongoDB) for flexible and scalable data storage.

\subsection{Database Collections}
\begin{itemize}
    \item \textbf{users:} Stores registered customer details (Name, Email, Password Hash).
    \item \textbf{rooms:} Stores hotel room inventory (Room Number, Floor, Type, Cost, Status).
    \item \textbf{bookings:} Stores reservations (Guest details, Room Number, Check-In/Check-Out Dates).
\end{itemize}

\section{Implementation Details}

\subsection{MongoDB Atlas Setup}
The database is hosted on MongoDB Atlas. Setting up involves:
\begin{enumerate}
    \item Creating a free cluster (Cluster0).
    \item Configuring \textbf{Network Access} to whitelist the application's IP address (resolving SSL handshake errors like \texttt{SSL alert number 80}).
    \item Obtaining the connection URI and configuring the \texttt{.env} file.
\end{enumerate}

\subsection{Application Initialization and Seeding}
To populate the database with initial room and booking data, a seeding script was implemented and executed using \texttt{npm run db:seed}. This automatically inserts mock data into the \texttt{rooms} and \texttt{bookings} collections.

\subsection{Features Implemented}
\begin{itemize}
    \item \textbf{Authentication Flow:} Secure Sign Up and Login utilizing \texttt{bcryptjs} and HTTP-only cookies.
    \item \textbf{Data Pre-filling:} Once authenticated, user details (Name, Email) are automatically fetched server-side and pre-filled into the Booking Form and Profile page.
    \item \textbf{Admin Dashboard:} Accessible at \texttt{/admin}. Provides CRUD operations for rooms. New rooms can be added using intuitive dropdown selections for room type and status.
\end{itemize}
